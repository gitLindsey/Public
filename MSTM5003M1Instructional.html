Security Fundamentals

Computer Security: Methods used to ensure a secure system that addresses authentication and access controls.

Network Security: Protection of multiple computers and devices that connect together.

Information Security: Focus on protecting the data itself from unauthorized access or alteration while maintaining availability.

Cybersecurity: Field covering availability of systems and information when needed.

Operational Model of Computer Security

Protection = Prevention + (Detection + Response)

Prevention

Firewalls, Access Controls, Encryption

Detection

IDS, Monitoring, Auditing

Response

Incident Response, Recovery, Forensics

Core Security Principles

Least Privilege

Minimum necessary rights and privileges

Separation of Privilege

Multiple pieces of information for access decisions

Fail-Safe Defaults

Default deny, explicit permission required

Economy of Mechanism

Keep solutions simple, eliminate unnecessary services

Complete Mediation

Verify every request through protection mechanisms

Open Design

Security shouldn't rely on mechanism secrecy

Defense in Depth

Multiple layered security mechanisms

Psychological Acceptability

Security measures must be user-friendly

Security Models

Bell-LaPadula Model (Confidentiality)

Simple Security Rule: "No read up"

*-Property: "No write down"

Focuses on data confidentiality in operating systems

Biba Model (Integrity)

Low-Water-Mark Policy: "No write up"

Ring Policy: "No read down"

Protects data integrity through access controls

Brewer-Nash Model (Chinese Wall)

Controls access based on conflict of interest rules

Prevents information flow when conflicts exist

Clark-Wilson Model

Transaction-based integrity model

CDI: Constrained Data Items

UDI: Unconstrained Data Items

Uses transactions and processes to maintain integrity
