Network Fundamentals

Network Architecture Analysis

Understanding network types and their security implications is fundamental to designing secure systems.

Network Types & Security Considerations

LAN

Scope: Limited geographic area

Security: Physical access control critical

Risks: Internal threats, broadcast storms

WAN

Scope: Geographically distributed

Security: Encryption mandatory

Risks: Man-in-the-middle attacks

VLAN

Scope: Logical segmentation

Security: Micro-segmentation

Risks: VLAN hopping attacks

Common Network Attack Vectors

  • ARP Spoofing: Malicious MAC address mapping
  • DNS Cache Poisoning: Corrupt DNS resolution
  • VLAN Hopping: Unauthorized network segment access
  • Wireless Interception: Unencrypted transmission capture

Protocol Security Analysis

TCP vs UDP

TCP: Reliable, ordered delivery

Security: Three-way handshake vulnerable to SYN flooding

UDP: Connectionless, fast

Security: No connection state to exploit, but no built-in reliability

IPv4 vs IPv6

IPv4: 32-bit addressing

Security: NAT provides basic hiding

IPv6: 128-bit addressing, built-in IPsec

Security: Larger attack surface, NDP vulnerabilities

Critical Insight: Protocol misconfigurations remain one of the most exploited attack vectors. Understanding the security implications of each protocol layer is essential for defense.

Mitigations: Port security, dynamic ARP inspection

Routers

Function: Layer 3 routing

Vulnerabilities: Route hijacking, ACL bypass

Virtualization Security

Virtualization Challenges:
  • VM Escape: Breaking out of virtual machine isolation
  • VM Sprawl: Unmanaged virtual machine proliferation
  • Shared Resource Attacks: Side-channel attacks between VMs
  • Container Security: Kernel sharing introduces new attack vectors

Defense Strategies

  • Hypervisor Hardening: Regular patching, minimal surface
  • Network Segmentation: Micro-segmentation between VMs
  • Resource Isolation: CPU, memory, and I/O controls
  • Monitoring: VM lifecycle and behavioral analysis

Data Security Controls

Data Loss Prevention (DLP) combines multiple technologies to protect sensitive information across its lifecycle.

At Rest

In Transit

TLS/SSL, VPN, secure protocols

In Use

Application controls, DRM, monitoring

Authentication Protocol Analysis

Kerberos

Type: Ticket-based

Strengths: Mutual authentication, SSO

Weaknesses: Single point of failure (KDC)

RADIUS

Type: Client-server

Strengths: Widely supported, accounting

Weaknesses: Shared secrets, UDP-based

TACACS+

Type: Client-server

Strengths: Granular authorization, TCP

Weaknesses: Cisco proprietary

Scenario-Based Assessment

Scenario: You discover that an employee's laptop has been accessing internal servers at unusual hours and downloading large amounts of data.

Incident Response Considerations:

  1. Immediate: Isolate the device, preserve evidence
  2. Investigation: Analyze logs, network traffic, user behavior
  3. Containment: Revoke access, change credentials
  4. Recovery: Assess data exposure, implement additional controls
Key Learning: This scenario highlights the importance of continuous monitoring, user behavior analytics, and having an incident response plan ready.

Key Takeaways for Graduate Students

Critical Security Principles

  • Security by Design: Networks are not inherently secure. Security must be integrated from the architectural phase, not added as an afterthought.
  • Layered Defense: Infrastructure devices enhance performance but expand attack surfaces. Implement defense in depth with multiple security layers.
  • Authentication Evolution: Move beyond password-only systems toward multi-factor authentication and comprehensive identity management frameworks.
  • Perimeter Dissolution: Remote access technologies blur traditional network boundaries. Security must be endpoint-aware and policy-driven.
  • Continuous Monitoring: Static security configurations are insufficient. Implement continuous monitoring and adaptive security controls.
